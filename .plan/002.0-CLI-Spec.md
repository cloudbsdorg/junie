## 1. Overview

The Junie CLI is the primary interface for users to interact with the Junie coding agent. It supports both interactive mode (read-eval-print loop) and single-command execution.

```bash
# Interactive mode
junie

# Single command mode
junie "Fix the bug in auth.go"

# Pipe input
echo "Review my PR" | junie
```

## 2. Global Flags

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--help` | `-h` | Show help text | — |
| `--version` | `-v` | Print version | — |
| `--no-color` | — | Disable colored output | false |
| `--verbose` | — | Enable verbose logging | false |
| `--debug` | — | Enable debug output | false |
| `--config` | `-c` | Path to config file | `~/.junie/config.toml` |

## 3. Commands

### 3.1 `junie` — Interactive Mode

Starts an interactive REPL session where users can communicate with the agent.

```bash
junie [flags]
```

**Flags:**

| Flag | Description |
|------|-------------|
| `--context` | Initial context to include |
| `--task` | Pre-defined task to execute immediately |

**Example:**

```bash
$ junie
Welcome to Junie! (Type 'exit' or 'quit' to end)

> Build a REST API endpoint for user management
[Agent responds with implementation plan and code]

> exit
Goodbye!
```

---

### 3.2 `junie auth` — Authentication

Manages authentication to JetBrains AI Hub or custom LLM providers.

```bash
junie auth [subcommand]
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `junie auth login` | Authenticate with JetBrains Account (OAuth) |
| `junie auth logout` | Clear stored credentials |
| `junie auth status` | Show current authentication status |
| `junie auth set-key` | Configure BYOK API key |
| `junie auth list-providers` | List available LLM providers |

**Examples:**

```bash
# OAuth login with JetBrains Account
junie auth login

# Set custom API key (BYOK)
junie auth set-key --provider anthropic --key sk-ant-xxxxx

# Check authentication status
junie auth status
# Current auth: JetBrains Account (user@example.com)
# Provider: JetBrains AI Hub
```

## 4. Interactive Mode Commands

When running in interactive mode, the following special commands are available:

| Command | Description |
|---------|-------------|
| `exit` / `quit` / `q` | End the session |
| `/help` | Show available commands |
| `/install-github-action` | Set up GitHub Action integration |
| `/feedback` | Submit feedback to JetBrains |
| `/clear` | Clear conversation history |
| `/context` | Show current context |

## 6. Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | General error |
| 2 | Authentication error |
| 3 | Configuration error |
| 4 | Network error |
| 5 | LLM provider error |

## 7. Examples

### Interactive Session

```bash
$ junie
> Build a user authentication module for my Go project

[Junie analyzes the project structure and creates auth.go with:
- User struct with ID, Email, PasswordHash fields
- JWT token generation and validation
- Login and Register handlers]

> Add unit tests for the auth module

[Junie creates auth_test.go with comprehensive test coverage]

> exit
```

### Single Command

```bash
$ junie "Explain what this function does: func (r *Repository) FindUserByID(id int) (*User, error)"
This function queries the database for a user with the specified ID.
It takes an integer id parameter and returns either a pointer to the
found User struct or an error if no user exists or the query fails.
```

### CI/CD Integration

```bash
# In GitHub Actions workflow
- name: Run Junie Code Review
  run: |
    junie "Review the changes in this pull request and highlight any potential issues"
```
