## 1. Overview

Junie can be configured as a GitHub Action to respond to repository events automatically. This enables AI-powered code review, issue response, and CI failure analysis.

---

## 2. GitHub Action Setup

### 2.1 Installation

Run the following command inside a Junie session:

```
/install-github-action
```

This generates a `.github/workflows/junie.yml` file:

```yaml
name: Junie AI Assistant

on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]
  workflow_run:
    types: [completed]
    workflows: [CI]

jobs:
  junie:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Junie
        run: |
          curl -fsSL https://junie.jetbrains.com/install.sh | bash
      - name: Run Junie
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          JUNIE_AUTH_TOKEN: ${{ secrets.JUNIE_AUTH_TOKEN }}
        run: |
          junie github respond --event ${{ github.event_name }}
```

### 2.2 Required Secrets

| Secret | Description |
|--------|-------------|
| `JUNIE_AUTH_TOKEN` | Authentication token from `junie auth login` |
| `GITHUB_TOKEN` | Auto-provided by GitHub Actions |

## 3. Webhook Events

Junie responds to the following GitHub events:

### 3.1 Issue Comments

Triggered when a user comments on an issue.

```yaml
on:
  issue_comment:
    types: [created]
```

**Example usage:**
```
@junie Can you explain why the build is failing?
```

### 3.2 PR Review Comments

Triggered when a user comments on a pull request.

```yaml
on:
  pull_request_review_comment:
    types: [created]
```

### 3.3 CI Workflow Completion

Triggered when a CI workflow completes.

```yaml
on:
  workflow_run:
    types: [completed]
```

**Example:**
```
@junie The CI failed. Can you analyze the error logs?
```

## 4. Permissions

The GitHub Action requires the following permissions:

```yaml
permissions:
  contents: read      # Read repository contents for context
  issues: write       # Respond to issues and comments
  pull-requests: write # Respond to PR reviews and comments
  statuses: write     # Post commit status updates
```

## 5. Usage Examples

### 5.1 Issue Response

```bash
# User comments on issue:
# "@junie The login endpoint returns 500 on POST /api/users"

# Junie responds with:
# "Looking at the error, it appears the UserService.Insert() method 
# is called with a nil pointer. In user_handler.go:42, we need to 
# check if the user object is nil before calling Insert()."
```

### 5.2 PR Review

```bash
# User creates a PR and requests review:
# "@junie review"

# Junie analyzes the diff and comments:
# "I found a few issues:
# 1. SQL injection vulnerability in query building (line 45)
# 2. Missing error handling in auth.go:67
# 3. Consider using prepared statements for the user query"
```

### 5.3 CI Failure Analysis

```bash
# CI workflow fails and user comments:
# "@junie ci failed"

# Junie:
# "The test_authentication[username=test] test is failing.
# The mock setup at line 123 doesn't match the actual API.
# Here's the fix..."
```
