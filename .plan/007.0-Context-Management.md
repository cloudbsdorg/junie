## 1. Overview

Context management is critical for Junie to understand the codebase and provide relevant responses. The context manager handles repository indexing, relevant code retrieval, and context window optimization.

---

## 2. Repository Indexing

On first run in a repository, Junie builds an index:

```bash
$ junie
> Indexing repository...
✓ Indexed 1,234 files (42 MB)
✓ Detected: Go project (go.mod found)
✓ Detected: Git repository
✓ GitHub remote: github.com/jetbrains/junie
```

### 2.1 Index Structure

| Component | Description |
|-----------|-------------|
| File tree | Hierarchical file structure |
| Symbol index | Functions, types, variables |
| Git history | Recent commits, branches |
| Dependencies | Package relationships |

---

## 3. Context Retrieval

When a user submits a task, Junie retrieves relevant context:

```
User: "Fix the authentication bug"

1. Parse task → keywords: "authentication", "bug"
2. Query file index → matching files
3. Retrieve file contents
4. Include in prompt
```

### 3.1 Retrieval Methods

- **Exact match**: Filename or path
- **Content search**: Full-text search
- **Symbol lookup**: Function/type name
- **Dependency analysis**: Import relationships

---

## 4. Context Window Management

LLMs have finite context windows. Junie optimizes context usage:

| Strategy | Description |
|----------|-------------|
| Truncation | Remove middle of large files |
| Summarization | Replace verbose code with summary |
| Prioritization | Include most relevant sections |

---

## 5. File Filtering

Files are filtered based on:

- **Extension**: Source files preferred
- **Size**: Large files truncated
- **Binary**: Skipped by default
- **Generated**: `.git`, `node_modules`, `vendor` excluded

### 5.1 Explicit Inclusion

Users can force-include files:

```
> Review the minified app.js file
```
