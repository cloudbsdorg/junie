# Junie — Glossary and Terminology

**Author:** JetBrains  
**Date:** 2026-04-30  

---

## Core Terms

### Agent
An autonomous software program that can perceive its environment, make decisions, and take actions to achieve specific goals. In Junie, the agent receives natural language instructions and translates them into development tasks.

### LLM (Large Language Model)
A neural network trained on vast amounts of text data to generate human-like text. Junie supports multiple LLM providers.

### LLM-Agnostic
A design principle where the system is not tightly coupled to a specific LLM provider. Junie achieves this through provider adapters that expose a common interface.

### BYOK (Bring Your Own Key)
A feature allowing users to configure their own API keys from supported LLM providers instead of using the default JetBrains service.

### Context Window
The maximum amount of text (measured in tokens) that can be passed to an LLM in a single request. Junie manages context to stay within these limits.

### Tool Use
The ability of an LLM to invoke external functions or operations (file read/write, git commands, shell execution) to accomplish tasks.

---

## Junie-Specific Terms

### Session
A conversation between the user and Junie. Sessions maintain state and allow Junie to understand context across multiple interactions.

### Memory Store
Persistent storage that allows Junie to retain information across sessions, such as user preferences and project-specific context.

### Context Manager
The component responsible for determining which files and code snippets are relevant to the current task and should be included in the prompt.

### Provider Adapter
A software component that provides a uniform interface to different LLM providers, handling authentication, request formatting, and response parsing.

### Tool Executor
The component that runs tools on behalf of the agent, with safety guardrails to prevent destructive operations.

---

## GitHub Integration Terms

### GitHub Action
A CI/CD workflow automation tool that can trigger Junie in response to GitHub events.

### Webhook
An HTTP callback that GitHub uses to notify Junie of events like issue comments, PR reviews, or CI failures.

---

## Platform Terms

### Platform Triplet
The combination of OS and architecture used to identify binary distributions (e.g., `darwin-aarch64`, `linux-x86_64`, `windows-amd64`).

### Release Channel
A category of releases indicating stability and update frequency:
- **Stable**: Monthly, production-ready releases
- **EAP** (Early Access Program): Bi-weekly, feature-preview releases
- **Experimental**: Weekly, bleeding-edge releases
- **Nightly**: Daily, internal testing releases
