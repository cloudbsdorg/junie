## 1. Overview

Junie uses a tool-use system to interact with the filesystem, run shell commands, and execute git operations. This enables the agent to accomplish real-world development tasks beyond just generating text.

---

## 2. Tool Categories

| Category | Description |
|----------|-------------|
| File I/O | Read, write, edit files |
| Git | Commit, branch, diff, merge |
| Shell | Execute commands |
| Search | Grep, glob, find |
| Web | Fetch remote content |

---

## 3. Tool Execution

Tools are called by the LLM when needed:

```json
{
  "tool": "read_file",
  "args": {
    "path": "src/auth.go",
    "offset": 1,
    "limit": 100
  }
}
```

Response:

```json
{
  "tool": "read_file",
  "result": "package auth\n\nfunc Login...",
  "error": null
}
```

## 4. Safety Guardrails

### 4.1 Destructive Operations

Commands that modify files require confirmation:

```
> Delete all test files
Junie: This will delete 47 files. Do you want to proceed? (yes/no)
```

### 4.2 Shell Execution

Shell commands are sandboxed:
- No interactive prompts
- Timeout after 60 seconds
- No environment variable access
- Limited PATH

### 4.3 Path Restrictions

- Cannot access files outside repository
- Symlinks are resolved
- Special paths blocked: `/etc`, `/var`, `~/.ssh`

## 5. Available Tools

| Tool | Description |
|------|-------------|
| `read_file` | Read file contents |
| `write_file` | Write/create files |
| `edit_file` | Apply targeted edits |
| `list_dir` | List directory contents |
| `glob` | Find files by pattern |
| `grep` | Search file contents |
| `git_*` | Git operations |
| `shell` | Execute shell commands |
| `web_fetch` | HTTP GET requests |
