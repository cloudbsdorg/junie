## 1. Overview

Security is a primary concern for Junie. This document outlines the security measures in place and best practices for users.

---

## 2. Data Privacy

### 2.1 Local Processing

- **File access**: All file operations occur locally on the user's machine
- **Prompt data**: Only text prompts are sent to LLM providers
- **Code remains local**: Source code never leaves the user's environment (except for GitHub integration when explicitly requested)

### 2.2 Telemetry

- Usage telemetry is opt-out
- No sensitive code is transmitted
- See [Telemetry document](./010.0-Telemetry.md) for details

---

## 3. Credential Handling

| Credential Type | Storage | Protection |
|----------------|---------|------------|
| OAuth tokens | OS Keychain | Platform encryption |
| API keys | OS Keychain | Platform encryption |
| GitHub tokens | GitHub Actions secrets | Encrypted at rest |

### 3.1 Best Practices

1. Use OS keychain instead of environment variables when possible
2. Never commit credentials to git
3. Rotate API keys periodically

---

## 4. Code Execution Safety

### 4.1 Sandboxing

Shell commands executed by Junie are:
- Non-interactive (no stdin)
- Timeout-limited (60s default)
- Environment-variable isolated
- PATH-restricted

### 4.2 Confirmation Prompts

Destructive operations require explicit confirmation:
- File deletion
- Force push
- Database operations
- Network requests outside scope

---

## 5. Network Security

- All API calls use HTTPS (TLS 1.2+)
- Certificate validation enabled
- Proxy support via `JUNIE_HTTP_PROXY`
- No self-signed certificate fallback

---

## 6. Vulnerability Reporting

If you discover a security vulnerability, please report it via:

- **GitHub**: [Security Advisories](https://github.com/jetbrains/junie/security/advisories)
- **Email**: security@jetbrains.com

For critical issues, please use our [bug bounty program](https://www.jetbrains.com/bug-bounty/).
