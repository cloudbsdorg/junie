## 1. Overview

Junie uses a provider adapter pattern to support multiple LLM providers transparently. This allows users to switch between providers without changing the core agent logic.

## 3. Provider Adapters

Each adapter implements the common interface:

```go
type LLMProvider interface {
    // Send a chat completion request
    Chat(ctx context.Context, req ChatRequest) (*ChatResponse, error)
    
    // Stream chat completion
    ChatStream(ctx context.Context, req ChatRequest) (*ChatStreamReader, error)
    
    // Get available models
    ListModels(ctx context.Context) ([]Model, error)
}
```

---

## 4. Request/Response Format

### 4.1 Chat Request

```go
type ChatRequest struct {
    Model    string            // Model identifier
    Messages []Message        // Conversation history
    System   string            // System prompt
    Tools    []ToolDefinition // Available tools
    Options  ChatOptions       // Temperature, max_tokens, etc.
}

type Message struct {
    Role    string // "user", "assistant", "system"
    Content string
}
```

### 4.2 Chat Response

```go
type ChatResponse struct {
    Content     string
    FinishReason string // "stop", "length", "tool_use"
    ToolCalls   []ToolCall
    Usage       Usage // Token usage statistics
}
```

## 5. Model Selection

Models can be specified via:

1. **Config file**: `[default] model = "claude-sonnet-4-20250514"`
2. **Environment variable**: `JUNIE_MODEL=claude-sonnet-4-20250514`
3. **Per-request**: In code, when calling the provider

### 5.1 Default Models by Provider

| Provider | Default Model |
|----------|---------------|
| JetBrains AI Hub | Provider's default |
| Anthropic | claude-sonnet-4-20250514 |
| OpenAI | gpt-4o |
| Google | gemini-2.5-pro |
| xAI | grok-2 |
| OpenRouter | varies |

## 6. Rate Limiting and Backoff

All providers implement retry logic with exponential backoff:

```
Attempt 1 → Immediate
Attempt 2 → Wait 1 second
Attempt 3 → Wait 2 seconds
Attempt 4 → Wait 4 seconds
Attempt 5 → Wait 8 seconds (max)
```

Rate limit errors (HTTP 429) trigger automatic backoff.

---

## 7. Error Handling

| Error Type | Handling |
|------------|----------|
| Network error | Retry with backoff |
| Rate limit | Wait and retry |
| Authentication error | Prompt re-auth |
| Model unavailable | Fall back to default model |
| Context length exceeded | Truncate or summarize context |
