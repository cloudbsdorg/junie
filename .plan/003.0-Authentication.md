## 1. Overview

Junie supports multiple authentication methods to accommodate different user preferences and organizational requirements. The authentication system is designed to be flexible while maintaining security.

---

## 2. Authentication Methods

### 2.1 JetBrains Account (OAuth)

The default authentication method uses JetBrains Account through OAuth 2.0.

**Flow:**

```
User → Junie CLI → JetBrains OAuth → JetBrains Account → Access Token
                                                              ↓
                                               JetBrains AI Hub API
```

**Commands:**

```bash
# Initiate OAuth login
junie auth login

# Browser opens to JetBrains login page
# After successful login, token is stored securely
```

**Token Storage:**

- Tokens are stored in platform-specific secure storage:
  - **macOS**: Keychain
  - **Linux**: libsecret (GNOME Keyring / KDE Wallet)
  - **Windows**: Credential Manager

---

### 2.2 API Key Authentication

Users can provide their own API key for the default JetBrains AI Hub service.

```bash
# Set API key
junie auth set-key --key YOUR_API_KEY

# Verify authentication
junie auth status
```

---

### 2.3 Bring Your Own Key (BYOK)

Junie supports using API keys from third-party LLM providers.

**Supported Providers:**

| Provider | Environment Variable | Config Key |
|----------|---------------------|------------|
| Anthropic | `ANTHROPIC_API_KEY` | `providers.anthropic.api_key` |
| OpenAI | `OPENAI_API_KEY` | `providers.openai.api_key` |
| Google | `GOOGLE_API_KEY` | `providers.google.api_key` |
| xAI | `XAI_API_KEY` | `providers.xai.api_key` |
| OpenRouter | `OPENROUTER_API_KEY` | `providers.openrouter.api_key` |
| GitHub Copilot | `GITHUB_TOKEN` | `providers.copilot.token` |

**Configuration:**

```bash
# Using CLI
junie auth set-key --provider anthropic --key sk-ant-xxxxx

# Using config file (~/.junie/config.toml)
[providers.anthropic]
api_key = "sk-ant-xxxxx"
default_model = "claude-sonnet-4-20250514"
```

**Switching Providers:**

```bash
# View available providers
junie auth list-providers

# Switch provider
JUNIE_PROVIDER=anthropic junie

# Or within config
[default]
provider = "anthropic"
```

## 3. Token Management

### 3.1 Token Refresh

OAuth tokens are automatically refreshed when they expire.

### 3.2 Token Revocation

```bash
# Logout (revokes token)
junie auth logout
```

### 3.3 Token Verification

```bash
$ junie auth status
JetBrains Account: user@example.com
Provider: JetBrains AI Hub
Status: Authenticated (expires in 30 days)
```

---

## 4. Provider Configuration

### 4.1 Configuration File

Location: `~/.junie/config.toml`

```toml
[default]
provider = "jetbrains"
model = "default"  # Provider-specific model name

[providers.anthropic]
api_key = "sk-ant-xxxxx"
default_model = "claude-sonnet-4-20250514"
max_tokens = 8192

[providers.openai]
api_key = "sk-xxxxx"
default_model = "gpt-4o"
max_tokens = 4096
```

---

## 5. Security Considerations

1. **No Plain Text Storage**: API keys and tokens are never stored in plain text
2. **Platform Security**: Uses OS keychain/credential manager
3. **Memory Safety**: Tokens are cleared from memory after use
4. **No Logging**: Tokens are redacted from all log output
5. **HTTPS Only**: All API communication uses TLS
